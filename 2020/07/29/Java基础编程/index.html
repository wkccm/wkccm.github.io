Java基础编程

第一章 Java语言概述

Java语言的主要特性：强制面向对象、分布式、健壮、解释型、体系结构中立、原生支持多线程

Java的核心机制：Java虚拟机、垃圾回收机制

第二章 基本语法

2.1 关键字与保留字

关键字：被Java语言赋予了特殊含义，用做专门用途的字符串，所有字母都小写

保留字：Java尚未使用，但是以后版本可能会使用

2.2 标识符

标识符的定义规则：

  • 由26个英文字母大小写，0-9，_或$组成
  • 数字不能开头
  • 不可以使用关键字和保留字，但可以包含
  • Java严格区分大小写，长度无限制
  • 标识符中不能包含空格

Java中的名称命名规范：

  • 包名：多单词组成时所有字母都小写
  • 类名、接口名：多单词组成时，所有单词首字母大写
  • 变量名、方法名：多单词组成时，第一个单词首字母小写，第二个单词开始每个单词首字母大写
  • 常量名：所有字母大写

2.3 变量

Java的整数类型占用的存储空间：byte 1 short 2 int 4 long 8

Java浮点数类型：float 4 double 8

char字符：\u 用Unicode值来表示字符型常量

布尔类型：只允许取值true和false，无null，不可以使用数字来代替

自动类型转换：

  • char、byte、short转int转long转float转double
  • 多种数据混合运算时，自动将所有数据转换成容量最大的数据类型
  • byte、char、short不会相互转换，互相计算时转成int
  • boolean类型不能与其他数据类型运算
  • 任何基本数据类型的值与String进行连接运算时，将自动转为String类型

强制类型转换：

  • 自动类型转换使用时加上强制转换符（）
  • boolean类型不可以转换
  • 字符串不能直接转换为基本数据类型，但可以调用基本类型对应的包装类

2.4 运算符

算术运算符注意事项：

  • 对负数取模，可以把模数负号忽略不计，但是被模数是负数则不可忽略，取模运算的结果不一定是整数
  • 对于除号，整数除会舍弃小数部分
  • 注意输出时涉及到String的自动类型转换

逻辑运算符：

  • 注意逻辑与&、逻辑或|与短路与&&、短路或||的区别

位运算符：

  • “>>>”被移位的二级制最高位无论是1还是0，空缺位都用0补

三元运算符：

  • 表达式1与2必须为同种类型

运算符的优先级：

  • 只有单目运算符、三元运算符和赋值运算符是从右向左运算的

2.5 程序流程控制

顺序结构、分支结构、循环结构

分支结构：

  • if-else语句
  • switch-case语句：
    • switch表达式的值必须是：byte、short、char、int、枚举、String
    • case中必须是常量
    • 各case中必须不同
    • break跳出switch语句块
    • dafault任选，当没有匹配的case时，执行default

循环结构：for、while、do-while

  • 标号语句必须紧接在循环的头部

第三章 数组

3.1 数组的概述

  • 数组是引用数据类型，数组中的元素可以是任何数据类型
  • 创建数组对象会在内存中开辟连续空间，数组名中引用的是首地址
  • 数组长度一旦确定，就不能更改
  • 可以通过下标直接调用

3.2 一维数组的使用

声明方式：type var[]或type[] var

栈中是局部变量，堆中是new出来的结构，如对象、数组

3.3 多维数组的使用

多维数组的初始化：

  • int 【】【】arr=new int【3】【2】
  • int 【】【】arr=new int【3】【】
  • int 【】【】arr=new int【】【】{ {1,2}，{3,4}，{5,6} }

第四章 面向对象编程（上）

4.1 面向过程与面向对象

面向对象的三大特征：封装、继承、多态

4.2 类和对象

类的语法声明：修饰符 class 类名{属性声明；方法声明}

4.3 对象的创建与使用

创建对象的语法： 类名 对象名=new 类名（）；

使用 对象名.对象成员 的方式访问对象成员

内存解析：

  • 堆：存放对象实例，数组
  • 栈：存储局部变量、对象引用，方法执行完自动释放
  • 方法区：用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码

匿名对象：不定义对象的句柄，而直接调用对象的方法，语法： new X（）.xxx（）；

匿名对象使用的情况：

  • 如果一个对象只需要进行一次方法调用，就可以使用匿名对象
  • 我们经常将匿名对象作为实参传递给一个方法调用

4.4 属性

语法格式： 修饰符 数据类型 属性名 = 初始化值；

4.5 方法

语法格式： 修饰符 返回值类型 方法名（参数类型 形参，……）{方法体 return 返回值；}

方法的注意事项：

  • 方法被调用一次，就会执行一次
  • 没有具体返回值的情况，返回值类型用关键字void表示，那么方法体中可以不使用return
  • 定义方法时，方法的结果应该返回给调用者，交由调用者处理
  • 方法中只能调用方法或属性，不可以在内部定义方法

4.6 再谈方法

重载：同一个类中，允许存在一个以上的同名方法，只要参数个数或者参数类型不同即可

可变个数的形参：

  • 格式：方法名（参数的类型名…参数名）
  • 可变参数的个数：0，1，多个
  • 可变个数的形参的方法与同名方法构成重载
  • 可变参数方法的使用与方法参数部分使用数组是一致的
  • 方法的参数部分有可变形参，需要放在形参声明的最后
  • 最多声明一个可变个数形参

值传递与引用传递

4.7封装与隐藏

修饰符 类内部 同一个包 不同包的子类 同一个工程
private yes
default yes yes
protected yes yes yes
public yes yes yes yes

class的权限修饰符只可以用public和default

4.8 构造器

  • 与类有相同的名称
  • 不声明返回值类型
  • 不能被static、final、synchronized、abstract、native修饰，没有return语句返回值
  • 构造器的作用：创建对象，给对象进行初始化
  • 语法：修饰符 类名（参数列表）{初始化语句；}
  • 每个类至少有一个构造器
  • 默认构造器的修饰符与所属类的修饰符一致
  • 一旦显式定义构造器，系统就不再提供默认构造器
  • 一个类可以创建多个重载的构造器
  • 父类的构造器不可被子类继承

赋值的先后顺序：

  • 默认初始化
  • 显式初始化
  • 构造器中初始化
  • 通过“对象.属性”或“对象.方法”的方式赋值

4.9 this的使用

注意事项：

  • 使用当前类的成员变量或成员方法可以加this
  • 形参与成员变量同名时，调用需要添加this
  • 使用this访问属性和方法时，若在本类中未找到，会从父类中查找
  • this可以作为一个类中构造器相互调用的特殊格式
  • 可以在类的构造器中使用“this（形参列表）”的方式，调用本类中重载的其他构造器
  • 不能通过this调用自身构造器
  • 如果一个类中声明了n个构造器，则最多有n-1个构造器使用了this（形参列表）
  • this（形参列表）必须在构造器首行
  • 在类的一个构造器中，最多只能声明一个this（形参列表）

4.10 package和import的使用

package指明文件中定义的类所在的包，语法：package 顶层包名.子包名；

import为使用定义在不同包中的Java类，需用import语句来引入指定包下所需要的类，语法：import 包名.类名；

如果在代码中使用不同包下的同名的类，就需要使用类的全类名的方式指明调用的是哪个类

如果已经导入某包下的类，且需要使用该包的子包下的类的话，仍然需要导入

import static组合的使用：调用指定类或接口下的静态的属性或方法

第五章 面向对象编程（中）

5.1 继承性

语法：class A extends B A是B的子类

Java支持单继承和多层继承，不允许多重继承

5.2 方法的重写

方法的重写：对父类中的方法进行改造，子类的方法将覆盖父类的方法

  • 子类重写的方法必须和父类被重写的方法具有相同的方法名称、参数列表
  • 返回值类型不能大于父类被重写的方法的返回值类型
  • 方法的访问权限不能小于父类被重写的方法的访问权限
  • 子类不能重写父类中声明为private权限的方法
  • 子类方法抛出的异常不能大于父类被重写方法的异常
  • 子类与父类中同名同参数的方法必须同时声明为非static的，或者同时声明为static的（不是重写）。因为static方法属于类，子类无法覆盖父类的方法

5.3 访问控制

5.4 super

在Java类中使用super来调用父类中的指定操作

  • 子类中所有的构造器默认都会访问父类中空参数的构造器
  • 当父类中没有空参数的构造器时，子类的构造器必须通过this或super语句指定调用本类或父类中相应的构造器，同时只能二选一，且必须放在构造器的首行
  • 如果子类构造器中既未显式调用父类或本类的构造器，且父类中又没有无参的构造器，则编译出错

super与this对比：

  • this先访问本类，再访问父类，super直接访问父类，两者都必须放在首行

5.5 子类对象实例化过程

5.6 多态性

对象的多态性：父类的引用指向子类的对象

Java的引用变量有两个类型：运行时类型和编译时类型

编译时，看左边；运行时，看右边

编译时用父类引用调用子类特有的属性和方法会产生错误

方法声明的形参为父类类型，可以使用子类的对象作为实参调用方法

虚拟方法调用：子类与父类有同名同参数的方法，父类方法为虚方法，编译期无法确定，以运行时为准

重载VS重写：

  • 重载，是指允许存在多个同名方法，早绑定/静态绑定
  • 重写，同名同参数，晚绑定，静态绑定

成员方法具有多态性，成员变量不具备多态性

x instanceof A：检验x是否是A的对象，返回boolean值

5.7 Object类的使用

==和equals的区别：

  • ==既可以比较基本类型，也可以比较引用类型，对于基本类型就是比较值，对于引用类型就是比较内存地址
  • equals在未重写的情况下默认是==，但一般会重写，重写后一般会比较类中相应属性是否都相等

5.8 包装类的使用

装箱：包装类使得一个基本数据类型变成了类

拆箱：将数字包装类中内容变为基本数据类型

第六章 面向对象编程（下）

6.1 static

static可修饰属性、方法、代码块、内部类

  • 随着类的加载而加载
  • 优先于对象存在
  • 修饰的成员，被所有对象共享
  • 访问权限允许时，可不创建对象，直接被类调用

在static方法内部，只能访问static修饰的属性和方法

static内部不能有this或super

static方法不能被重写

6.2 理解main方法的语法

6.3 代码块

对Java类或对象进行初始化

执行顺序与显示初始化平级

可以用static修饰，静态代码块先于普通代码块执行

6.4 final

final可用来修饰类、变量、方法

final标记的类不能被继承，不能被子类重写

final标记的变量即为常量，只能被赋值一次

6.5 抽象类与抽象方法

用abstract修饰的类为抽象类，修饰的方法为抽象方法

  • 抽象类不能被实例化
  • 抽象方法只有声明，没有实现
  • 含有抽象方法的类必须被声明为抽象类
  • 不能用abstract修饰变量、代码块、构造器
  • 不能用abstract修饰私有方法、静态方法、final的方法、final的类
  • 抽象类的子类必须重写父类的抽象方法，并提供方法体

6.6 接口

接口是抽象方法和常量值定义的集合

接口interface，继承接口时implements

  • 先写extends，后写implements
  • 一个类可实现多个接口，接口也可以继承其他接口
  • 接口冲突时，要指明接口名

6.7 内部类

  • 外部类访问成员内部类的成员，需要用“内部类.成员”或“内部类对象.成员”
  • 内部类可以直接使用外部类的所有成员，包括私有的数据
  • 当想要在外部类的静态成员部分使用内部类时，可以考虑内部类声明为静态的
  • 局部内部类可以写在方法内、代码块内、构造函数内
  • 可以使用外部方法的局部变量，但是必须是final的，由声明周期不同所致
  • 局部内部类可以使用外部类的成员，包括私有的
  • 局部内部类不能用static修饰

匿名内部类不能定义任何静态成员、方法和类，只能创建一个实例

  • 匿名内部类必须继承父类或实现接口

第七章 异常处理

7.1 异常概述与异常体系结构

异常事件可分为两类，error和exception，error是Java虚拟机无法解决的严重问题，exception是因编程错误或偶然的外部因素导致的一般性问题，可以使用针对性的代码进行处理

异常分为运行时异常和编译时异常

7.2 常见异常

7.3 try-catch-finally

try中选定捕获异常的范围，catch中是对异常对象进行处理的代码，每个try语句块可以伴随一个/或多个catch语句

finally：无论代码块中是否发生了异常，catch语句是否执行，catch中是否有return，finally都会执行

7.4 throws

如果一个方法可能生成某种异常，但是并不能确定如何处理，则方法声明中用throws语句可以声明抛出异常的列表，throws后面的异常类型可以是方法中产生的异常类型，也可以是它的父类

重写方法不能抛出比被重写方法范围更大的异常类型

7.5 手动抛出异常

生成异常类对象，然后通过throw语句实现抛出

7.6 用户自定义异常类

  • 用户自定义异常类一般都是RuntimeException的子类
  • 通常需要编写几个重载的构造器
  • 需要提供serialVersionID
  • 自定义的异常通过throw抛出
